Сценарий 1: Добавление нового питомца
    Метод: POST /pet
    Описание: Добавление нового питомца в магазин.
    Ожидаемый результат: Успешное добавление питомца возвращает статус 200 и объект питомца с присвоенным ID.

Сценарий 2: Получение информации о питомце по ID
    Метод: GET /pet/{petId}
    Описание: Получение информации о питомце по его ID.
    Ожидаемый результат: Получение существующего питомца по ID возвращает статус 200 и корректные данные питомца.

Сценарий 3: Обновление информации о питомце
    Метод: PUT /pet
    Описание: Обновление существующего питомца.
    Ожидаемый результат: Успешное обновление питомца возвращает статус 200 и обновлённые данные.

Сценарий 4: Поиск питомцев по статусу
    Метод: GET /pet/findByStatus
    Описание: Поиск питомцев по определённому статусу (например, available).
    Ожидаемый результат: Поиск по статусу возвращает статус 200 и список питомцев с указанным статусом.

Сценарий 5: Удаление питомца по ID
    Метод: DELETE /pet/{petId}
    Описание: Удаление питомца из магазина по его ID.
    Ожидаемый результат: Успешное удаление питомца возвращает статус 200.

Сценарий 6: Авторизация пользователя
    Метод: GET /user/login
    Описание: Вход пользователя в систему.
    Ожидаемый результат: Успешная авторизация возвращает статус 200 и сообщения о сессии.

Сценарий 7: Создание нового пользователя
    Метод: POST /user
    Описание: Регистрация нового пользователя.
    Ожидаемый результат: Успешная регистрация пользователя возвращает статус 200.
